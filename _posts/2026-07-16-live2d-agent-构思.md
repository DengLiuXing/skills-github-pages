# 告别巨石架构：打造 AI 时代的“VSCode 式”桌面宠物
最近，我花了不少精力完整部署了 airi 和 NagaAgent 这两个开源桌面宠物项目。平心而论，它们的技术令人咂舌，但在部署完成的那一刻，我感到了一种强烈的疲惫和不认同。
这两个项目为了追求“完全自主”、“长时记忆”、“复杂情绪状态机”，堆砌了极其庞大且互相耦合的架构。如果我们的核心诉求仅仅是——在桌面上有一个能语音交互、能触发特殊动作、能根据对话切换状态的“陪伴感”实体，这种大而全的巨石架构不仅显得冗余，反而拉高了维护和二次开发的门槛。
于是，我决定放弃它们，开始构思一套全新的设计理念。
## 灵感来源：大脑外包，本地只做画板
我的第一个灵感来源于现代 Agent（如 OpenClaw、Hermes CodeX 等）能力的飞跃。
既然现代 AI 智能体已经具备了极强的意图理解、逻辑推理甚至动作决策能力，我们为什么还要在本地维护一套复杂的“行为树”和“状态机”呢？
传统的桌宠开发思路是：本地写一堆复杂的逻辑判断，如果用户说了什么，触发什么动作。而在 AI 时代，这个思路应该被彻底颠覆：**“大脑外包给现代 Agent，本地外壳只负责 IO 和渲染”**。
我们只需要让架构满足最基础的三件事：持续监听（STT）、语音播报（TTS）、Live2D 模型控制。最多再加一个互动组件栏，通过简单的指令交互，让 AI 来驱动桌宠的喜怒哀乐。
## 理念延伸：从“轻量外壳”到“VSCode 式微内核”
当思路走到这一步时，我的想法发生了一次关键的延伸。
如果只是为了轻量，那它仍然只是一个定制化的应用。但我希望这个项目不仅仅是给自己用的，而是能成为一个繁荣的生态底座。我想到了 **VSCode**。
VSCode 之所以能一统前端开发工具市场，不是因为它内置了多少功能，而是因为它提供了一个极其优秀的微内核和插件系统。
因此，我的最终设计理念确立为：**主程序是一个“空壳”，只提供基础画布和插件接口；所有的核心能力，包括 STT、TTS、Agent 对接，甚至交互组件栏，全部下沉为插件。**
当然，在 Live2D 的渲染上，我会汲取 airi 和 Naga 的现有成果进行复刻。我知道 Live2D 的渲染依然会消耗大量 GPU，所以我不打算从“极致省电”这方面来宣传这个项目，而是把核心优势聚焦在**架构的开放性与易用性**上。
## 浅谈架构与三大核心优势
基于上述理念，整个系统的架构可以极其精简，核心只包含三部分：微内核（管理插件生命周期）、全局事件总线（用标准 JSON 流进行发布/订阅通信）、以及极简的 SDK。
为了让这个底座具有真正的生命力，它必须具备以下三大优势：
### 1. 极低的上手门槛（1天即可创作）
任何一个计科大学生，只要懂一点基础的 JavaScript 和网络请求，就能在一天内写出一个可用的插件。不需要理解复杂的底层框架，只需要声明一个 `manifest.json`，然后在 `index.js` 里实现 `activate` 函数，监听或发送事件即可。
### 2. AI 驱动的插件开发（SDK 友好）
这是最面向未来的设计。我们提供一份结构化、极度清晰的 API/SDK 手册。即使一个开发者完全不懂我们主程序的内部实现，他也可以把这份手册直接丢给 AI，让 AI 根据手册轻易开发出新的交互插件。
### 3. 无缝衔接 Agent 生态与具身化终端体验（杀手级优势）
因为主程序只认标准的事件流，这意味着后端的“大脑”可以是任意现代 Agent 框架。无论是 Dify 编排的工作流，还是 Coze 上发布的机器人，甚至是一个简单的 GPT Function Calling 脚本，都可以直接作为一个“Agent 插件”接入。
更重要的是，这种架构让桌宠彻底突破了“玩具”的定位，变成了一个**具身化的 Agent 集成终端**。
传统的 Agent 终端往往是冷冰冰的命令行或聊天框，普通人难以亲近。而通过我们的项目，Agent 获得了可视化的外壳和拟人化的交互能力。用户通过语音或互动栏下达指令，背后的 Agent 调用各种工具（查天气、读文件、控制系统）并将结果通过桌宠的语音和动作反馈出来。这种“有温度的 AI 助手”形态，不仅能直接复用现有的海量 Agent 插件生态，更能以极低的使用门槛快速吸引大量潜在的非技术用户。
## 写在最后
airi 和 Naga 是在造“全家桶”，用户必须接受它们规定好的一整套逻辑；而我希望造的是一个“底座”。
小白可以直接安装主程序，将其作为自己专属的 AI 终端运行；老手可以关掉所有默认插件，自己花半小时写一套对接本地大模型和自训练音色的插件组；而广大的 AI 开发者，可以直接把他们辛苦开发的 Agent 工具挂载到这个桌宠上，瞬间获得一个“活的”前端。
放弃巨石，回归简单，把创造力交还给插件生态和 Agent 网络，这才是 2024 年以后，桌面宠物该有的样子。
